# Content Creation Guide for Dawar Azhar's Engineering Blog

This guide provides comprehensive instructions for creating high-quality engineering articles for the blog, whether generated by AI agents or written manually.

## üìù Article Structure Template

### Required Frontmatter Format
```markdown
---
title: "Specific, SEO-friendly title (50-60 characters)"
description: "Compelling meta description that summarizes the article's value (150-160 characters)"
publishDate: YYYY-MM-DD
tags: ["primary-category", "specific-technique", "tools-used", "application-area", "difficulty-level"]
draft: false
image: "/images/posts/[appropriate-category-image].jpg"

```

### Standard Article Structure
1. **Hook Opening** (1-2 paragraphs)
   - Start with a compelling problem, question, or industry challenge
   - Establish why this topic matters to engineers
   - Preview the value readers will gain

2. **Overview/What You'll Learn** (1 paragraph)
   - Brief outline of key concepts covered
   - Prerequisites or background knowledge needed
   - Expected outcomes after reading

3. **Main Content Sections** (3-5 H2 sections)
   - Each section should focus on one key concept
   - Use descriptive headers like "Understanding the Physics" not "Section 1"
   - Include practical examples and real-world applications

4. **Implementation/Practical Example** (Required)
   - Step-by-step procedures
   - Code snippets with explanations
   - Troubleshooting common issues

5. **Results/Performance Analysis** (When applicable)
   - Benchmarks, metrics, or validation data
   - Comparison with alternative approaches
   - Discussion of limitations

6. **Conclusion & Next Steps**
   - Summarize key takeaways
   - Suggest related topics for further learning
   - Encourage reader engagement

## ‚úçÔ∏è Writing Style Guidelines

### Tone & Voice
- **Professional but approachable**: Like explaining to a fellow engineer over coffee
- **Authoritative**: Demonstrate deep technical knowledge
- **Practical**: Always connect theory to real applications
- **Encouraging**: Help readers feel confident to try new techniques

### Technical Writing Standards
- **Explain jargon**: Define technical terms on first use
- **Use active voice**: "We calculated the Reynolds number" not "The Reynolds number was calculated"
- **Be specific**: Use exact values, tool versions, and parameter settings
- **Include units**: Always specify units for measurements and calculations
- **Cite sources**: Reference equations, standards, and research papers

### Article Length Guidelines
- **Tutorial articles**: 1200-2000 words
- **Concept explanations**: 800-1500 words
- **Tool reviews**: 600-1200 words
- **Quick tips**: 400-800 words

## üíª Code & Technical Examples

### Code Block Standards
```python
# Always specify the language for syntax highlighting
# Include necessary imports at the top
import numpy as np
import matplotlib.pyplot as plt

def calculate_reynolds_number(velocity, length, kinematic_viscosity):
    """
    Calculate Reynolds number for flow analysis.
    
    Args:
        velocity (float): Flow velocity in m/s
        length (float): Characteristic length in m
        kinematic_viscosity (float): Kinematic viscosity in m¬≤/s
    
    Returns:
        float: Reynolds number (dimensionless)
    """
    return velocity * length / kinematic_viscosity

# Example usage with realistic values
Re = calculate_reynolds_number(10.0, 0.1, 1.5e-5)  # Re ‚âà 66,667
print(f"Reynolds number: {Re:.0f}")
```
